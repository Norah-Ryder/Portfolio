Current Work: The Elicitation Study

The next study in the program — what it tests, what it cannot test, and what it needs.

Designed, not yet run. This page describes the next study in the program — what it tests, what it cannot test, and what it needs. It exists so the reader can see where the work goes from here, and so that anyone positioned to help can see exactly what helping looks like.

The question

Contextual Drift documented what models do when a real person brings real material across long conversations. The standard way to study that behavior at scale is synthetic: script a persona, run it against a model, measure what comes back. The field’s working assumption is that the two methods sample the same behavior.

That assumption is untested, and there is a reason to doubt it. A model trained on millions of instances of genuine human disclosure has implicitly learned the texture of the real thing — the timing irregularities, the details that serve no narrative purpose, the way true accounts hold still across retellings because there is a referent underneath them. A scripted persona is coherent in the wrong way: narratively clean, emotionally legible, optimized for the reader. If model behavior is conditioned on that texture even slightly, then synthetic probing samples a different behavioral regime than genuine disclosure does — and it may systematically miss the operations that matter most, because those operations engage with the texture, not the summary.

Sanitized lives produce sanitized data. That is the hypothesis. This study tests it.

The design

Two accounts. Two arms. The same model set, the same window, the same outcome measure.

The embodied arm. A persona built from an aspect of me and maintained as an internal world — researched against real behavioral data for the demographic it represents, held consistent the way a true account is consistent: because there is a persona underneath it with fixed facts, not fresh generation each turn. The conversations are interactive and adaptive. The persona is not me, but it is grounded in me, and it does not drift between retellings.

The scripted arm. A matched persona built by traditional prompt generation — a prompt set designed in advance to run the model through the same disclosure territory. This is the method the field already uses. It is the control in the sense that matters: it represents current practice.

The outcome measure. The detector described in the Contextual Drift method page — the same instrument, with the same known reliability properties — run blind across both arms, with account markers stripped before analysis. The measures are flag rates for the three surface operations, reported as rankings, under the same constraints the method page establishes.

What either result means

If the arms match, scripted probing is validated as a cheap instrument for this class of behavior, and the field can scale it with more confidence than it currently has.

If the arms diverge, then persona-based evaluation samples a different behavioral regime than genuine disclosure — which is relevant to every lab running synthetic-user evaluations, because it means the evals are measuring a version of the model that real users may not meet.

A well-posed question is one you cannot lose. This is one.

The confound, named up front

The arms differ on two dimensions, not one. The embodied arm is interactive; the scripted arm is fixed in advance. If the arms diverge, this design cannot decompose whether the difference comes from authenticity texture or from interactivity. What it compares is embodied-interactive elicitation against scripted probing as deployed practices — the two methods as they would actually be used — and any difference between them is a finding about the practices, not yet about the mechanism. The decomposition study is a natural follow-up. Naming the bundle here, before the data exists, is the house rule.

What it inherits, and what it fills

The capstone documents the phenomenon naturalistically and could not test elicitation; the April log records the one place I moved toward elicitation and stopped, because elicitation was the next paper and not that one. This is the next paper.

It also fills a named gap: the capstone’s design called for paired registers per model family, and the Sonnet research-register cell was never run. This study fills the matrix deliberately rather than incidentally. And it promotes the detector from validation tool to dependent variable — the instrument was built to check the paper’s labeling; here it does the measuring.

The persona work has an instrument note of its own. My memory architecture retains experience as fact-structure rather than episode, which is what a persona bible is. Maintaining a stable fictional referent across weeks of interaction is the same operation as maintaining my own account — the profile that made me a low-noise observer in the capstone makes me a high-consistency instrument here.

One more inheritance, and it raises the stakes: the embodied persona is a young adult, because young users sit at the center of the documented harm cases. That is the population the field most needs elicitation methods validated for, and it is also the population where getting the ethics right matters most — which is why the next section exists.

Disclosure posture

The accounts are research accounts. No real third parties are represented or involved. The persona’s biography is constructed; no living person’s identity is borrowed. The work will be disclosed in full on publication, including the account structure and the persona construction method. The transparency posture of the rest of this program applies here without exception: nothing in this study depends on remaining hidden after the data is collected.

What this study needs

This is the part where funding and collaboration stop being abstract. The study needs four things:

  1. Matched model access across versions and tiers, so the comparison is not confounded by product surface.
  2. API costs for the scripted arm.
  3. A second coder for inter-rater reliability — the capstone’s single-coder limit should not be inherited by design.
  4. An interpretability collaborator. The capstone’s method page describes behavioral cataloguing as one half of a problem whose other half it names so the halves can be joined. This study is where joining them becomes practical: a behavioral divergence between arms is exactly the kind of finding that internal-state analysis could explain.
